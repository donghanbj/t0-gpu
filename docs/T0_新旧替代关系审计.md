# T0 编译器全面审计：新旧替代关系

> 基于 28 个源文件的逐行依赖分析 | 2026-03-24

---

## 一、核心发现

> [!CAUTION]
> T0 编译器存在**系统性的"内部已进化、外部未切换"问题**。SSA 优化、SSA 寄存器分配、SSA 编译路径的代码都已写好，但生产调用链仍走旧路径。

---

## 二、逐模块替代关系

### 1. 优化管线：`opt_passes.rs` × `ssa_ir.rs`

| 维度 | 实际状态 |
|------|---------|
| **旧版函数** | `dead_code_elimination`、`constant_fold`、`algebraic_simplify`、`copy_propagate`、`common_subexpr_elim`、`instruction_combine` — **全部标记 `#[deprecated]`**，代码仍在文件中但已不再被调用 |
| **新版函数** | `ssa_ir::dce_mach_func`、`constant_fold_mach_func`、`algebraic_simplify_mach_func`、`copy_propagate_mach_func`、`cse_mach_func_domtree`、`instruction_combine_mach_func`、`licm_mach_func`、`optimize_waitcnt_mach_func`、`schedule_mach_func` |
| **实际调度** | `opt_passes::optimize()` **已切换到 SSA 路径**！4 阶段：Phase A (6 SSA pass) → Phase B (loop_unroll + strength_reduce) → Phase C (iterative AlgSimp+DCE) → Phase D (waitcnt + scheduling) |
| **结论** | ✅ **优化管线已完成切换**。`opt_passes.rs` 是 SSA 优化的调度壳，旧函数是死代码。可安全删除 deprecated 函数（~600 行） |

---

### 2. 寄存器分配：`regalloc.rs` × `ssa_regalloc.rs`

| 维度 | 实际状态 |
|------|---------|
| **旧版** | `regalloc::allocate()` — 线性扫描，interval-sorted |
| **新版** | `ssa_regalloc::allocate_ssa()` — SSA 活性分析 + best-fit + spill 支持 |
| **切换开关** | `T0Kernel.use_ssa_regalloc` 默认 `false`（[compile.rs:65](file:///home/geis/tprime_optimized/rust-port/tprime-brain/src/OCPA/opensource/src/t0/compile.rs#L65)） |
| **谁调用 `set_ssa_regalloc(true)`？** | **没有任何代码调用过** |
| **依赖关系** | `ssa_regalloc.rs` 输出类型是 `regalloc::RegAlloc`，`asm_emitter.rs` 也用 `RegAlloc` |
| **结论** | ⚠️ **生产路径走旧版**。新版写好了但没启用。`regalloc.rs` 的 `RegAlloc` 结构体是公共接口，不能删 |

---

### 3. 编译路径：`compile()` × `compile_via_ssa()`

| 维度 | 实际状态 |
|------|---------|
| **旧路径** | `BlockKernel::compile(target)` — 在 `block_dsl.rs` 内联实现，直接 emit `T0Kernel` ops |
| **新路径** | `BlockKernel::compile_via_ssa(target)` — 在 `block_dsl_to_ssa.rs` 实现，走 BNode→TileFunc→tile_ssa_lower→T0Kernel |
| **ignis 调用** | **15 处全部用 `kb.compile(Target::GFX1100)`**（rmsnorm×3, silu×2, add×3, embedding×2, fusion×3, psi_activation×1, shape_ops×1） |
| **谁用 `compile_via_ssa`？** | **仅 `block_dsl_to_ssa.rs` 自带的 16 个测试** |
| **结论** | ⚠️ **生产路径走旧版**。SSA 编译路径只在测试中验证 |

---

### 4. 前端层：`dsl.rs` × `block_dsl.rs` × `block_ir.rs`

| 维度 | 实际状态 |
|------|---------|
| **`dsl.rs` Op 枚举** | 40+ 变体（Scale/ReLU/GEMM/OCPA 等）。被 `dsl_lower.rs` 使用 |
| **`dsl_lower.rs`** | Op → `math.rs`/`gemm_gen.rs` 模板映射。被 `gpu_tests` 3 个测试 + ignis `gpu_context.rs` 间接引用 |
| **`block_dsl.rs`** | Triton 风格 BNode DAG，103KB。**ignis 生产使用** |
| **`block_ir.rs`** | 类型安全 T0Builder 图构建器。被 `block_lower.rs` + `block_jit.rs` + `gpu_tests` 3 个测试使用 |
| **`dsl.rs` 共用类型** | `CompiledKernel`/`DType`/`KernArgMeta`/`KernArgType`/`KernArgValue` — **被 6+ 模块引用，不可移除** |
| **结论** | `dsl.rs` 中 Op 枚举是迭代残留，但共用类型是活的基础设施。`block_ir.rs` 未接入统一管线 |

---

### 5. 降低层：`dsl_lower.rs` × `block_lower.rs` × `tile_ssa_lower.rs`

| 维度 | 实际状态 |
|------|---------|
| **`dsl_lower.rs`** | Op→T0Kernel。调用 `math.rs` 模板和 `gemm_gen`。仅被 `gpu_tests` 使用 |
| **`block_lower.rs`** | block_ir 图→CompiledKernel。调用 `math.rs` + `gemm_gen` + `tile_ir`。被 `block_jit` + `gpu_tests` 使用 |
| **`tile_ssa_lower.rs`** | TileFunc→T0Kernel。调用 `tile_ir` + `gemm_gen`。是统一管线的目标降低器 |
| **结论** | `dsl_lower.rs` 和 `block_lower.rs` 是旧链路的残留 |

---

### 6. 内核模板：`math.rs`

| 维度 | 实际状态 |
|------|---------|
| **大小** | ~250KB, 7800+ 行 |
| **ignis 直接调用** | `cross_entropy`×4, `ocpa_attention`×5, `tensor`×2, `bf16_matmul`×1, `tape`×1 |
| **`dsl_lower` 调用** | 所有 elementwise/reduce/特殊内核 |
| **`block_lower` 调用** | elementwise 融合、reduce |
| **通过 `ensure_kernel_t0` 消费** | ignis 直接调用 `math::xxx()` → 返回 T0Kernel → `ensure_kernel_t0` 编译加载 |
| **结论** | ✅ **math.rs 是核心生产代码，不可移除**。它不是"被替换"了，而是作为内核模板库同时被新旧链路使用 |

---

### 7. `schedule.rs` — 调度 trait

| 维度 | 实际状态 |
|------|---------|
| **`GFX1100Schedule` 使用者** | `math.rs`×10, `dsl_lower.rs`×8, `block_lower.rs`×3, `gpu_tests`×1 |
| **`Schedule` trait** | 持有 WG size 等硬件参数，被 `math.rs` 的函数签名使用 |
| **结论** | ✅ 活跃使用中，但仅服务于 `math.rs` 模板路径。统一管线（`tile_ssa_lower`）不使用它 |

## 四、可执行的清理优先级

| 优先级 | 操作 | 风险 | 收益 |
|-------|------|------|------|
| **P0** | 删除 `opt_passes.rs` 中 6 个 `#[deprecated]` 旧函数体（~600 行死代码） | 零风险（已不被调用） | 减少混淆 |
| **P1** | 从 `dsl.rs` 提取共用类型到 `types.rs`，Op 枚举留在 `dsl.rs` 标记 legacy | 低风险（re-export 兼容） | 明确边界 |
| **P2** | 将 `compile.rs` 的 `use_ssa_regalloc` 默认改为 `true` | **中风险**（需全量测试验证正确性） | 启用 SSA RegAlloc |
| **P3** | ignis 从 `kb.compile()` 切换到 `kb.compile_via_ssa()` | **高风险**（SSA 编译路径可能有未覆盖 case） | 统一管线 |
| **P4** | 移动 `dsl.rs`/`dsl_lower.rs`/`block_ir.rs`/`block_lower.rs` 到 `legacy/` | 低风险（re-export） | 物理隔离 |
