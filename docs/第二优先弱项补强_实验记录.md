## 方法与结果

### 已实施的修改

| 项 | 文件 | 修改内容 | 验证 |

### 审计发现（不需修改）

| 项 | 原审计结论 | 实际状态 |
|----|-----------|---------|
| SSA regalloc coalesced_group | 不理解 8-align | ✅ 已有 VRegAlloc → group_id/size/index 支持 |
| dsl_lower kernarg_size 硬编码 40 | 影响正确性 | ✅ dsl_lower.rs 已不存在 |
| deprecated 死代码 600 行 | 需清理 | ✅ 已在之前清理完毕 |

| SumReduce 未实现 | CPU fallback | CPU 足够快（loss 标量），GPU 需 block_dsl reduce 原语 |
| emit_store_phase_masked | regalloc 冲突 | 根因：循环内 VReg 过度分配 + raw_asm。非关键（padded Y 可用） |

### ISA Verifier 新增检查

| 检查 | 类型 | 说明 |
|------|------|------|
| WMMA 操作数对齐 | warning | pre-regalloc 虚拟 VReg 不需 8-align |
| global_store vscnt | warning | 未 wait_vscnt 可能丢失 store |
| LDS/SMEM lgkmcnt | 追踪 | pending 计数器 |
| VReg ≥ 256 | warning | 可能超 VGPR 限制 |
| SBASE 奇数对齐 | error | SMEM 要求偶数 SGPR 对 |

## 结论

1. 审计文档有 ~50% 的发现已过时——代码在迭代中已自然解决
2. DispatchPool `Mutex` 解除了跨线程限制，后续可并行测试
3. ISA verifier pre/post regalloc 区分很重要——虚拟 VReg 语义不同于物理寄存器
4. block_dsl 缺少 `reduce_add` 原语，GPU sum 需要该原语后实现
